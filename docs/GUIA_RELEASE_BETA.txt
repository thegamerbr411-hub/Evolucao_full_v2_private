GUIA DE RELEASE FINAL E BETA - T-EVO

1) O QUE O APP FAZ
T-Evo e um app de evolucao fisica com foco em treino, nutricao, coach, social e acompanhamento continuo.
Ele permite registrar uso real, gerar insights, acompanhar performance e exportar diagnostico beta para auditoria posterior.

2) O QUE CADA AREA/ABA FAZ
Home:
- Resumo do dia, status de metas, atalhos para treino e nutricao.

Treino:
- Hub de treinos, execucao de series, progresso, descanso, detalhes e video de exercicio.

Nutricao:
- Registro de alimentos/refeicoes, agua, busca de alimentos, fluxo de OCR/foto e registro manual.

Coach:
- Chat de suporte com contexto do usuario e recomendacoes.

Social:
- Amigos, desafios, ranking, feed e progresso colaborativo.

Perfil:
- Dados da conta, versao/build, sessao, beta/export, logout/relogin.

Admin:
- Acoes administrativas (XP/missoes), cadastros locais, logs e controle de acesso de conta.

3) COMANDOS E ROTINAS IMPORTANTES
npm start
- Inicializa app no Expo.

npm run android
- Executa build/execucao Android de desenvolvimento.

npm run release:apk
- Gera APK de release local para validacao manual.

npm run qa:full:visual:quick
- Ciclo rapido de validacao visual automatizada.

npm run qa:visual:audit:5x
- Repete ciclo visual varias vezes para detectar regressao.

npm run qa:prod:check
- Check de consistencia para producao.

npm run qa:cleanup
- Limpeza de artefatos temporarios de QA.

11) GOOGLE LOGIN - CHECKLIST DE DIAGNOSTICO
- Validar client IDs corretos (Android e Web) no app/config.
- Validar SHA-1 e SHA-256 no Firebase/Google.
- Validar package name Android.
- Validar consent screen publicada e com escopos corretos.
- Validar retorno ao app e persistencia da sessao.

Erros comuns:
- invalid_request custom URI scheme
- custom scheme nao permitido no client errado
- code challenge invalido

Acao recomendada:
- manter fluxo nativo do provider Google do Expo sem forcar redirectUri manual inconsistente.

12) OCR E NUTRICAO - CHECKLIST
- Testar foto da tabela nutricional.
- Confirmar parse de calorias, carbo, proteina, gordura, sodio/acucar quando disponivel.
- Confirmar fracao de consumo (total/parcial).
- Confirmar fallback manual sem perder campos.
- Confirmar que dados entram no export beta com contexto de origem (OCR/manual).

13) CONTROLE DE ACESSO (ANTI-REPASSSE)
No Admin:
- Bloquear conta por UID/e-mail.
- Desbloquear conta.
- Revogar sessao.

Efeito esperado:
- Conta bloqueada nao loga novamente.
- Sessao revogada invalida token anterior.
- Mesmo com APK compartilhado, acesso passa a depender do estado da conta no backend.
